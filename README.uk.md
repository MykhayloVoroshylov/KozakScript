KozakScript — це експериментальна багатодіалектна мова програмування, натхненна C++, Python та Pascal.

Проект спочатку створювався як цікава та освітня ідея: мова програмування, яка дозволяє людям вивчати та писати код, використовуючи синтаксис та термінологію, ближчі до їхньої рідної мови. З часом проєкт перетворився з простого прототипу на функціональну інтерпретовану мову з підтримкою пакетування виконуваних файлів, кількома діалектами синтаксису та генерацією автономних додатків.

KozakScript наразі підтримує:
- Український синтаксис
- Синтаксис української кирилиці
- Російський синтаксис
- Синтаксис російської кирилиці
- Англійський синтаксис
- Символічний синтаксис

Мова все ще перебуває в активній розробці, і постійно додаються нові функції, покращення синтаксису та модулі.

# 🚀 Інсталяція

Завантажте останню версію та розпакуйте файли будь-де у вашій системі.

Для запуску програм KozakScript або пакетних виконуваних файлів не потрібна окрема установка Python.

---

# ▶️ Запуск програм

Використовуйте виконуваний файл інтерпретатора для запуску файлів `.kozak`:

```bash
KozakScriptInterpreter.exe path\to\program.kozak
```

Приклад:

```bash
KozakScriptInterpreter.exe hello.kozak
```

# 📦 Об'єднання виконуваних файлів у пакет

KozakScript містить вбудований пакетер, здатний пакувати програми `.kozak` в окремі виконувані файли Windows.

Об'єднані програми:
- містять необхідне середовище виконання внутрішньо
- не потребують встановлення Python
- можуть включати додаткові ресурси та модулі

## Базове використання

```bash
kozak_bundler.exe my_program.kozak
```

---

## 📁 Adding Additional Files

You can embed external files into the final executable.

### Format

```bash
kozak_bundler.exe my_program.kozak --add-data source_path;destination_name
```

### Приклад

```bash
kozak_bundler.exe game.kozak --add-data config.json;config.json
```

Можна додати кілька файлів:

```bash
kozak_bundler.exe game.kozak ^
--add-data config.json;config.json ^
--add-data assets/sprite.png;sprite.png ^
--add-data sounds/hit.wav;hit.wav
```
