# KozakScript

**[English 🇬🇧](README.md) | [Українська 🇺🇦](README.uk.md)**

Я программист-ентузиаст и решил создать собственный прототип языка программирования. Будучи украинцем, я счёл интересной идею создать язык, отражающий мою культуру, который при необходимости мог бы использоваться профессиональными программистами или дать украиноязычным людям возможность, которой у меня лично никогда не было: изучать программирование, используя более знакомые термины и слова. Именно поэтому я не остановился на синтаксисе одной программы HelloWorld, а пошёл дальше, сделав язык по-настоящему функциональным.

Синтаксис в основном похож на C++ и Python с элементами Pascal. Однако у него есть свои уникальные особенности, такие как поддержка четырёх диалектов (украинского, русского, английского и символического). Вы можете посмотреть примеры синтаксиса в файле test.kozak или немного подождать: как только я закончу версию 1.0, я предоставлю .txt-файл со всеми правилами синтаксиса и, возможно, видео на YouTube с их объяснением. Кроме того, примеры программ можно найти в папке «Examples».

Это ещё не полная версия — я всё ещё улучшаю её, добавляю новые функции и исправляю ошибки. Большая часть уже функциональна и готова к использованию, так что можете начинать экспериментировать! :)

Все файлы с пометкой «test» используются для тестирования функций. Пожалуйста, не изменяйте их; вы можете создавать свои собственные файлы для экспериментов (или профессиональной разработки, если хотите :) ).

## 🔧 Требования

Раньше KozakScript требовал наличия Python на компьютере конечного пользователя, но теперь это не требуется.

Однако для упаковки программ в исполняемые файлы **ResourceHacker** необходим **только если вы хотите установить собственный значок**.

### Требования для использования сборщика
- **ResourceHacker** (необязательно — требуется только для установки собственных значков)
Пользователи должны установить его вручную.

___

### ▶️ Как запустить

### **Вариант 1 — Запуск с помощью интерпретатора**
Используйте предоставленный интерпретатор для запуска любого файла `.kozak`:

KozakScriptInterpreter.exe path\to\file.kozak

Убедитесь, что файл имеет расширение `.kozak`.

### **Вариант 2 — Упаковка в отдельный исполняемый файл**
Используйте встроенный упаковщик для упаковки вашей программы `.kozak` в отдельный исполняемый файл `.exe` для Windows.

1. Поместите файл `.kozak` в папку проекта.
2. Запустите инструмент **Bundler**.
3. (Необязательно) Если вам нужен собственный значок, поместите `ResourceHacker.exe` в каталог упаковщика.

Полученный `.exe` полностью переносим и *не* требует Python.

# 📦 Сборка

KozakScript включает в себя собственный упаковщик C++, который позволяет превратить любой файл `.kozak` в автономное приложение Windows `.exe`.

Для встроенного исполняемого файла Python **не требуется**.

### 🔨 Базовое использование

```bash
kozak_bundler.exe my_program.kozak
```
### 🎨 Собственная иконка

```bash
kozak_bundler.exe my_program.kozak --icon icon.ico
```

ПРИМЕЧАНИЕ: RessourceHacker должен быть установлен и находиться в том же каталоге, что и файлы сборки.

Если ResourceHacker.exe не найден, сборщик:

- выведет справочное сообщение с инструкциями по установке

- спросит, хотите ли вы продолжить установку без применения значка. Если ввести «y», сборка будет выполнена со значком по умолчанию, если ввести «n», сборка будет остановлена.

### 📁 Добавление данных

Формат:
```bash
kozak_bundler.exe my_program --add-data source_path;destination_name
```
Пример:
```bash
kozak_bundler.exe game.kozak --add-data config.json;config.json
```

Вы можете указать несколько записей --add-data:
```bash
kozak_bundler.exe game.kozak --add-data config.json;config.json --add-data assets/sprite.png;sprite.png --add-data sounds/hit.wav;hit.wav
```

📌 Полная комманда, пример:
```bash
kozak_bundler.exe program.kozak --icon logo.ico --add-data settings.cfg;settings.cfg --add-data levels.dat;levels.dat
```
Это позволит:

1. скопировать интерпретатор

2. встроить ваш скрипт .kozak

3. встроить ресурсы и связанные модули

4. встроить дополнительные файлы данных

5. применить пользовательский значок (если установлен ResourceHacker)

6. создать финальный .exe-файл в папке build_exe
